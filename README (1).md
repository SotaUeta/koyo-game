# 🍁 こうよう祭 学科対抗陣取りゲーム

豊田高専キャンパスを舞台にした、学科対抗の街づくり×陣取り位置情報ゲームです。
ブラウザだけで動く1ファイル構成（`index.html` + マップ画像）。データ共有は Firebase Realtime Database（無料枠）、公開は GitHub Pages（無料）を使います。

## フォルダ構成

```
koyo-game/
├ index.html   … ゲーム本体（設定もこの中）
├ campus.png   … キャンパスマップ画像
└ README.md    … このファイル
```

## セットアップ（所要 約15分）

### 1. Firebaseプロジェクトを作る

1. https://console.firebase.google.com にGoogleアカウントでログイン →「プロジェクトを追加」
2. プロジェクト名は自由（例: `koyo-game`）。Googleアナリティクスは不要なのでオフでOK
3. 左メニュー「構築」→「Realtime Database」→「データベースを作成」
   - ロケーションは `asia-southeast1` などでOK
   - セキュリティルールは「テストモードで開始」を選ぶ
4. 「ルール」タブを開き、以下に書き換えて「公開」（ログイン済みユーザーのみ読み書き可）

```json
{
  "rules": {
    ".read": "auth != null",
    ".write": "auth != null"
  }
}
```

> ⚠ このルールは「ログインしていれば何でも書ける」状態です。βテストには十分ですが、本番（こうよう祭当日）前にはユーザーごとの書き込み制限への強化を検討してください。

5. 左メニュー「構築」→「Authentication」→「始める」→「ログイン方法」タブで以下を有効化
   - **メール / パスワード** … 「有効にする」をON（メールリンクは不要）
   - **Google** … 有効にしてサポートメールを選択
6. Authentication →「設定」→「承認済みドメイン」に、公開先のドメイン（例: `ユーザー名.github.io`）が入っているか確認（無ければ追加。`localhost` は最初から入っています）
7. プロジェクト設定（⚙アイコン）→「全般」→「マイアプリ」→ ウェブアプリ（`</>`）を追加
8. 表示される `firebaseConfig = { ... }` をコピー

### 2. index.htmlに設定を貼る

`index.html` を開き、上のほうにある

```js
const firebaseConfig = {
  apiKey: "PASTE_YOUR_API_KEY",
  ...
};
```

を、コピーした自分の設定に丸ごと置き換えます。
**`databaseURL` が含まれているか確認**（無ければ Realtime Database の画面上部に表示されるURL `https://xxxx-default-rtdb....firebasedatabase.app` を追記）。

### 3. GitHub Pagesで公開する

1. GitHubで新しいリポジトリを作る（Public）
2. `index.html` `campus.png` `README.md` をアップロード（Web画面の「Add file → Upload files」でOK）
3. リポジトリの Settings → Pages → Branch を `main` / `(root)` にして Save
4. 数分後 `https://ユーザー名.github.io/リポジトリ名/` で公開される

このURLを友達に送れば**すぐ複数人でβテスト開始**できます。スマホのブラウザで開いてホーム画面に追加すると遊びやすいです。

## βテストの流れ

1. URLを開くとログイン画面（メール/パスワードで新規登録、またはGoogleでログイン）
   - ログインすると**どの端末・ブラウザからでも同じデータ**で遊べる
   - ログイン前に遊んでいた端末のデータは、初回ログイン時に自動でアカウントへ引き継がれる
2. 初回は名前入力＋陣営（M/E/I/C/A）選択。スターターとして自陣営の学科長（Aランク人材）1体・コイン5枚・エナドリ6本が配布される
3. マップのアイコンをタップ（**βテスト中は位置に関係なくタップ可能**）
   - 自陣営の建物 → 防衛／強化／配給
   - 敵・中立の建物 → 攻撃（最大6体選んで出撃、オートのターン制バトル）
4. ガシャは100円玉1枚（5分に1枚回復、最大5枚）

### データを全部リセットしたいとき

Firebaseコンソール → Realtime Database → データのルート（`players`・`buildings`・`rankings`）を削除。
誰かがページを開くと建物データは初期状態で自動再生成されます。
アカウント自体（Authenticationのユーザー）は残るので、次にログインすると新規プレイヤーとして登録画面から始まります。アカウントも消したい場合は Authentication → Users から削除してください。

## 本番（当日）に向けた切り替え

`index.html` 上部の設定を変えるだけです。

| 設定 | βテスト | 本番 |
|---|---|---|
| `BETA_MODE` | `true`（どこでもタップ可） | `false`（近くにいる時だけタップ可） |
| `TAP_RADIUS_M` | — | 建物ごとの許容距離(m)。GPS精度を考えると 80〜150 が目安 |
| `BUILDING_DEFS` の `lat`/`lng` | 仮の値のまま | **現地で計測して差し替え**（Googleマップで建物を長押しすると座標が出ます） |

## ゲーム仕様まとめ（実装済み）

- **陣営**: M(赤)・E(黄)・I(青)・C(緑)・A(水色)。各学科棟がメイン拠点
- **施設**: 図書館・第1/第2体育館・食堂・ものづくりセンター・プール・陸上競技場・テニスコート（最初は中立、HPを0にすると自陣営に。制圧時HP全回復）
- **防衛**: 建物に「引きこもり」を配置（1建物1人1体、倒されたら再配置可）。防衛で敵を倒すと配置した人に🍁こうよう
- **強化**: 素材で攻/守Lvアップ → 学科の倍率が上がる。🌾畑Lvアップ → その建物の配給量が増える。強化でも🍁がもらえる
- **配給**: 建物ごとに1時間に1回。エナドリ/菓子パン、食堂だけ🍱Aセット（全回復）。その建物の🌾畑Lvが高いほど量が増える（エナドリ/菓子パンは畑Lvと同じ個数、Aセットは畑Lvの半分）。クールダウン中も「次に何がいくつもらえるか」表示
- **学科の倍率**: 所有建物数・強化Lv・建物HPから自動計算。建物が攻撃されると下がる。倍率が高い学科の建物ほど素材を多く落とす
- **素材**: 🔩ネジ(M)・🔌回路(E)・💻コード(I)・💧水(C)・🏠模型(A)。その学科の建物から出やすい(60%)
- **バトル**: 最大6体・オートターン制・攻撃側先攻。引きこもり→建物の順に攻撃。拠点はHP無限（1未満にならず時間で回復、削ると倍率ダウン）。建物を殴るほうが素材は多い
- **属性**: 人材はガシャ時に🔥火・💧水・⚡雷・🪨土・🌀風・❄️氷のどれかを持つ。相性は「火に水、水に雷、雷に土、土に風、風に氷、氷に火が強い」の6すくみで、有利属性への攻撃はダメージ1.5倍（バトルログに「効果バツグン」表示）。属性追加前の人材は無属性（相性なし）
- **人材いちらん**: ランク順／攻撃力順／守り順／HP順で並び替え、属性で絞り込みができる
- **ガシャ**: F50% / C30% / B15% / A5%。ステータスに個体差±10%
  - HPは「ローマ字表記が短い」ほど高め、攻撃力は「画数が多い」ほど高め、の傾向で調整
- **ランキング**: 陣営（倍率順）＋陣営内（🍁こうよう順）
- **バチバチ表示**: 過去1分以内に戦闘があった建物のアイコンに⚡が出る

## 調整しやすいポイント（index.html内を検索）

- `UNIT_POOL` … 人材の追加・ステータス変更（先生の追加もここ）
- `GACHA_RATE` … 排出率
- `COIN_MS` / `RATION_MS` … コイン・配給の間隔
- `upCost()` … 強化コスト
- `BUILDING_DEFS` … 建物の追加、マップ上の位置(x,y は画像の%)

## 既知の割り切り（βテスト仕様）

- 同時に同じ建物を攻撃すると後勝ち（結果が一部上書きされることがある）
- チート耐性は弱い（ログインすれば他人のデータにも書ける）→ 本番前にルール強化を推奨

## ログイン機能について

- Firebase Authentication（メール/パスワード・Google）でログインし、AuthのUIDをプレイヤーIDとして使用
- **どの端末からでも同じアカウントでログインすれば同じデータ**で遊べる
- 旧バージョン（localStorageで端末識別）のデータは、その端末で初回ログインしたときに自動でアカウントへ引き継がれる（人材・素材・防衛配置・ランキングごと移行）
- ヘッダー右上の ⏻ ボタンでログアウト
- パスワードを忘れた場合は、ログイン画面の「パスワードを忘れた場合はこちら」で再設定メールを送信
